# ISモニタリング

IS(インサイドセールス)・マーケが獲得した初回アポイントの案件を週次でモニタリングするダッシュボードです。

**データはGoogleスプレッドシート([リード管理シート](https://docs.google.com/spreadsheets/d/1_TJRjTq8PRg0FTs36CbB5I45EKrccVEkpqWw2UO7Stc/edit)の `all_cases` タブ)から自動で読み込みます。** ページを開く/「最新データに更新」を押すたびに最新化されます。

対象メンバー(初回アポイント取得者):

| メンバー | 所属 | 注力フェーズ |
|---|---|---|
| 林我玖 | IS | P3(アカプラは01商談でも可のため、新規リード/アカプラの内訳も表示) |
| 奈良 | IS | P1 |
| 松村佳保 | マーケ | — |

## 見られるもの

- **獲得アポ総数**と、その案件が今 **P1〜P8 / Dead** のどこに何件あるか(メンバー別)
- **月次目標**に対する今月のアポ獲得数(目標は「設定」から変更可、初期値10件/月)
- **週次推移**(初回アポイント日ベースの新規アポ数、直近8週)
- **週次スナップショット**(フェーズ分布の推移。ページを開くたびに当週分を自動保存)
- **主担当別の案件数**(進行中 / 受注 / Dead の内訳つき)
- **Tier別の企業**と案件数
- 数字をクリックすると**案件一覧**(区分・アポ日・DEAD理由つき)が開き、案件名クリックで **Salesforce** の該当商談に飛びます
- デフォルトでは**継続案件・再計上案件を除外**(ヘッダーのチェックで含められます)

## 使い方

`index.html` をブラウザで開くだけです(サーバー不要)。GitHub Pages でホストする場合はリポジトリの Settings → Pages で公開してください。

### データ連携の仕組み

Google Sheets の gviz エンドポイント(`/gviz/tq?tqx=out:json&sheet=all_cases`)をブラウザから直接読みます。シートが「リンクを知っている全員が閲覧可」になっている必要があります。使用している列:

| 列 | 用途 |
|---|---|
| 初回アポイント取得者 | メンバーの判定(対象3名の行だけ集計) |
| フェーズ | `01-…`〜`08-Closed` → P1〜P8、`DEAD-…` → Dead |
| 初回アポイント日 | 週次・月次のアポ数集計 |
| 新規/継続 | `継続案件`/`再計上案件`はデフォルト除外(トグルで切替) |
| 顧客担当者氏名 | リード単位の案件数カウント(新規リード判定)に使用 |
| 主担当者 / 取引先名 / 案件 ID / 施策区分_2 / DEAD理由 / 案件種別 / 注力案件 | 集計・案件一覧の表示 |

同一案件IDの重複行は自動で1件にまとめます。接続に失敗した場合は、前回取得のキャッシュ → 手動取込データ → サンプル(`data/deals.js`)の順でフォールバックします。

### 新規リード / アカプラ(既顧客)の定義

どちらも**対象企業のリードのみ**を集計します(対象: 三菱電機・サントリーホールディングス・サントリー・サントリープロダクツ・サントリー食品インターナショナル・日本電気・ウーブン・バイ・トヨタ・野村不動産ホールディングス・野村不動産。「株式会社」を除いた完全一致で判定)。

- **新規リード**: 案件が1件しかないリード(取引先×顧客担当者)の案件
- **アカプラ(既顧客)**: 「新規/継続」が`既顧客新規案件`の案件

リードの案件数は all_cases 全行(担当・期間・継続を問わず)でカウントします。対象企業の変更は `index.html` の `TARGET_ACCOUNTS` を編集してください。

### マーケ(松村)への帰属:「エンプラ松村」ワード

案件のヒアリング情報等に **「エンプラ松村」** ワードが含まれる場合、マーケ起点のアポとして扱います(検出は列名に依存せず案件の全項目を横断。`アポイント時ヒアリング情報` 列にも自動対応)。

- 初回アポイント取得者が **林我玖 / 奈良和馬**(IS): 獲得アポは**本人に計上**し、松村カードに「📣 マーケ経由ISアポ ◯件」(26年6月以降計測)として**別掲**(松村の獲得アポには含めない)
- それ以外の取得者(松村本人を含む): **松村(マーケ)の獲得アポに計上**

キーワードの変更は `index.html` の `MKT_KEYWORD` を編集してください。

### Tier について

**`all_lead` タブの「備考」列**(`Tier1`〜`Tier4`、「Tier2クロスセル対象企業」等の表記ゆれも対応)から取引先名ごとに自動判定します。判定できない企業は「Tier未設定」にまとまります。手動で上書きしたい場合は、スプレッドシートに **`tier` という名前のタブ**(1列目: 取引先名、2列目: Tier)を作ると、そちらが優先されます。

### Salesforceリンクの設定

「設定」→「Salesforceドメイン」に自社の Lightning ドメイン(例: `https://xxxx.lightning.force.com`)を入力すると、案件一覧の案件名クリックで SF の商談画面が開きます。

### 月次目標の設定

「設定」からメンバーごとの月次目標(アポ獲得数)を変更できます(初期値: 10件/月)。
