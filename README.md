# ISモニタリング

IS(インサイドセールス)・マーケが獲得した初回アポイントの案件を週次でモニタリングするダッシュボードです。

対象メンバー(初回アポイント取得者):

| メンバー | 所属 | 注力フェーズ |
|---|---|---|
| 林我玖 | IS | P3(アカプラは01商談でも可のため、新規リード/アカプラの内訳も表示) |
| 奈良 | IS | P1 |
| 松村佳保 | マーケ | — |

## 見られるもの

- **獲得アポ総数**と、その案件が今 **P1〜P8 / Dead** のどこに何件あるか(メンバー別)
- **月次目標**に対する今月のアポ獲得数(目標は「設定」から変更可)
- **週次推移**(作成日ベースの新規アポ数、直近8週)
- **週次スナップショット**(フェーズ分布の推移。ページを開くたびに当週分を自動保存)
- **主担当別の案件数**(進行中 / 受注 / Dead の内訳つき)
- **Tier別の企業**と案件数
- 数字をクリックすると**案件一覧**が開き、案件名クリックで **Salesforce** の該当商談に飛びます

## 使い方

`index.html` をブラウザで開くだけです(サーバー不要)。GitHub Pages でホストする場合はリポジトリ設定で Pages を有効にしてください。

### 1. データの入れ方(2通り)

**A. 画面から取り込む(推奨・手軽)**
「CSV取込」ボタンから SF エクスポートの CSV を選択するか、スプレッドシートの範囲(ヘッダー行含む)をコピーしてそのまま貼り付けます。データはブラウザ(localStorage)に保存されます。

**B. ファイルで管理する(チームで共有したい場合)**
`data/deals.js` の `rows` 配列を実データに置き換えてコミットします。

> ⚠ 現在の `data/deals.js` は**サンプルデータ**です。

### 2. 必要な列

| 列名 | 必須 | 備考 |
|---|---|---|
| 初回アポイント取得者 | ✔ | 「林我玖」「奈良」「松村佳保」を含む行だけが対象になります |
| 案件名 | ✔ | |
| フェーズ | ✔ | `01-…`〜`08-Closed`、`DEAD-…` 形式(SFの値そのまま) |
| 主担当者 | 推奨 | 主担当別集計に使用 |
| 取引先名 | 推奨 | Tier別集計に使用 |
| 作成日 | 推奨 | 週次・月次集計に使用(`YYYY/MM/DD`) |
| 案件ID | 推奨 | SFへのリンクと重複行(候補者リスト分)の除去に使用 |
| Tier | 任意 | 例: `Tier1` `Tier2` |
| リード区分 | 任意 | `新規` / `既存` |
| 商談種別 | 任意 | `01商談` / `アカプラ` など |

同一案件IDの重複行(候補者リストで行が分かれているもの)は自動で1件にまとめます。

### 3. Salesforceリンクの設定

「設定」→「Salesforceドメイン」に自社の Lightning ドメイン(例: `https://xxxx.lightning.force.com`)を入力すると、案件一覧の案件名クリックで SF の商談画面が開きます。

### 4. 月次目標の設定

「設定」からメンバーごとの月次目標(アポ獲得数)を変更できます(初期値: 10件/月)。
